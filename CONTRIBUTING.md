# Contributing

Thanks for your interest in improving this project.

## Development workflow

1. Create a feature branch: `feature/<short-name>` (or `fix/<short-name>`)
2. Keep changes focused and well-scoped
3. Open a Pull Request (PR) with a clear description of what and why

## Commit style

Use clear, conventional commit messages when possible, for example:
- `feat: add new ingestion connector`
- `fix: handle malformed source payload`
- `docs: update sources and keywords`

## Testing

Run the project's test/lint steps once they exist in the repository. If there are no
automated checks yet, please add a minimal verification note in the PR description.
